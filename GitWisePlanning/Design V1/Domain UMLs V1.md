2. group the commits into repos
3. per repository iterate through each commit.
4. get a list of all filenames 


4. for each commit if the filename hasn't been seen before get the blob contents.

5. now have a dictionary of filenames to their blob contents which is the first iteration of it on that particular day - can be used for context for subsequent patch differences.

6. get a list of commits of the same filename, and then iterate over them and build a request model that has the filename, the list of patches in order with datetime and message associated.
